  1. Ange sidans språk i koden

    Sidans språk är angivit på toppen av html koden.

  2. Ange språkförändringar i koden

    Jag har skrivit in lang notering i de delar av koden som har annat språk än sidan.

  3. Använd inte enbart färg för att förmedla information

    Jag använder inte färg för att förmedla någon information på sidan.

  4. Använd tillräcklig kontrast mellan text och bakgrund

    Jag har tillräckligt kontrast mellan text och bakgrund för att man läsa den. Vissa ställen kan det bli lite bättre på men det är fortfarande inte svårt att läsa texten.

  5. Använd tillräckliga kontraster i komponenter och grafik

    Jag har tillräckligt med kontrast mellan de knappar jag har och bakgrunden, knappar är de enda komponenterna jag har på sidan. Den enda knappen som är svår att se är min easter egg knapp, som är gjord att vara svår att hitta.

  6. Beskriv med text allt innehåll som inte är text

    Jag har alt text på alla bilder jag har på webbsidan förutom den som ligger på framtids sidan då den är bara en del av bakgrunden och är där för att vara dekorativ. Jag har också skrivit in en titel på min svg drop down meny som beskriver vad den är.

  7. Erbjud alternativ till komplexa fingerrörelser

    Använder inga fingerrörelser utan webbsidan använder bara klickningar för att navigera och se innehåll.

  8. Gör det möjligt att hoppa förbi återkommande innehåll

    Jag har skrivit in en skip länk längst upp i headern som skippar till innehållet på sidan.

  9. Förmedla information, struktur och relationer i koden

    Jag använder korrekt typ av element för att visa den information jag vill visa på sidan. Till exempel ser jag till att använda h element för rubriker.

  10. Gör det möjligt att pausa, stänga av eller sänka ljud

    Jag har inget ljud på webbsidan.

  11. Gör det möjligt att justera tidsbegränsningar

    Jag har inga tidsbegränsningar på webbsidan.

  12. Gör det möjligt att pausa eller stänga av rörelser

    Jag har inga rörelser på webbsidan.

  13. Ge förslag på hur fel kan rättas till

    Jag har inget ställe på webbsidan där en användare kan göra något fel som jag kan ge feed back på.

  14. Ge möjlighet att ångra, korrigera eller bekräfta vid viktiga transaktioner

    Det finns inga transaktioner på webbsidan

  15. Gör det möjligt att ångra klick

    Har inga funktioner som behöver ångras på webbsidan. Enda knappar som finns är för att få mer info om projekt på projekt sidan och easter egg knappen för att ändra bakgrunden. Projekt info pop uppen kan man stänga med en stäng knapp eller escape knappen/ klicka utanför fönstret och easter egg bakgrunden ändras tillbaka när man laddar om sidan.

  16. Ha en meningsfull fokusordning

    Fokusordningen är bra och följer en förståelig struktur. Detta tack vare att jag skrivit de element som är interaktibla på sidan i en förståelig ordning.

  17. Markera tydligt vilket fält eller element som är i fokus

    När element är i fokus ändrar jag både storlek på texten, samt lägger till en röd border som gör det lättare att se vad som är i fokus.

  18. Märk upp vanliga formulärfält i koden

    Jag har inga formulärfält på webbsidan.

  19. Popup-funktioner ska kunna hanteras och stängas av alla

    Popup fönster kan stängas genom att klicka på krysset i rutan, klicka utanför fönstret och genom att klicka på escape. Samt kan man fokusera på krysset och klicka på enter för att stänga fönstret.

  20. Presentera innehållet i en meningsfull ordning för alla

    Informationen på sidan är skriven i den ordning den ska läsas i. Detta gör att alla kan läsa den I en meningsfull ordning.

  21. Se till att innehållet anpassas efter skärmens riktning

    Sidan anpassar de element som behövs till den storlek av skärm och orientering som används för att det ska gå att använda.

  22. Se till att det går att öka avstånd mellan tecken, rader, stycken och ord

    Sidan anpassar sig utifrån det avstånd som användaren vill ha genom att det finns utrymme för text att ta upp.

  23. Se till att hjälpmedel kan presentera meddelanden som inte är i fokus

    Webbsidan har inget som presenterar ett meddelande som behöver denna kriteria.

  24. Se till att koden validerar

    Validerade koden med W3C validation tool och fixade de saker som dök upp.

  25. Se till att markören inte fastnar vid tangentbordsnavigation

    Det går att navigera sidan med tangentbord och man fastnar inte på några ställen.

  26. Se till att skräddarsydda komponenter fungerar i hjälpmedel

    Det finns alt text till alla element som behöver det.

  27. Se till att text går att förstora utan problem

    Texten kan förstoras utan att sidan ser konstig ut då den anpassas.

  28. Se till att text på knappar och kontroller överensstämmer med maskinläsbara etiketter

    Det finns en arial-label på de länkar och knappar som behöver det.

  29. Skapa en flexibel layout som fungerar vid förstoring eller liten skärm

    Webbsidan anpassas utifrån den storlek av skärm som används, samt om skärmen ligger på sidan.

  30. Skapa kortkommandon med varsamhet

    Webbsidan använder inga kortkommandon.

  31. Skapa tydliga och klickbara fältetiketter/ledtexter

    Det finns inga inmatnings fält på webbsidan.

  32. Utför inga oväntade förändringar vid fokusering

    Det enda som händer när ett element fokuseras är att det får en border som hjälper en att se vilket element som är fokuserat, samt att det kan bli lite större för att enklare se det.

  33. Utför inga oväntade förändringar vid inmatning

    Webbsidan har inget element som detta påverkar.

  34. Utveckla systemet så att det går att hantera med enbart tangentbordet

    Hela webbsidan går att använda genom tangentbordet.

  35. Visa var ett fel uppstått och beskriv det tydligt

    Användaren kan inte göra något som kan bli fel på webbsidan. Det enda stället fel kan uppstå har ett felmeddelande.
